##CUSTOMR ORDER ANALYSIS USING POSGRESQL


Customer Order Analysis (PostgreSQL)
##Overview
This project focuses on analyzing customer, product, and order data using PostgreSQL. The goal is to extract meaningful business insights such as customer distribution, product performance, revenue trends, and order activity.
No external tools (Python, Excel, visualization libraries) were used—only SQL queries executed directly in PostgreSQL.

Objectives
The analysis addresses key business questions:
- Identify customers by region and name patterns.
- Evaluate product pricing and ordering trends.
- Track order activity across months and quarters.
- Measure customer growth and retention.
- Calculate revenue contributions by product categories.
- Detect pending shipments.


Key Insights
- Regional distribution of customers helps identify strong and weak markets.
- High-value products (>300) can be prioritized for premium customer segments.
- Monthly and quarterly order trends reveal seasonality and customer acquisition patterns.
- Revenue analysis by category highlights which product lines drive profitability.
- Pending shipment detection ensures operational efficiency.

 Technology Used
- Database: PostgreSQL
- Approach: Pure SQL queries (no Python, no visualization tools)
